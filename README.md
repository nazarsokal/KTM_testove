# Flight Telemetry Analyzer (KTM Testove)

Цей проєкт призначений для аналізу, обробки та 3D-візуалізації польотної телеметрії з MAVLink логів. 

Він поєднує:
* **Бекенд:** розроблений на платформі .NET 8
* **Обробка даних:** скрипти на Python
* **Фронтенд:** сучасний інтерфейс на React з використанням 3D-графіки

---

## ⚙️ Вимоги до системи

Для розгортання проєкту на вашому комп'ютері потрібно мати попередньо встановлені:
* Node.js
* Python 3.x
* .NET 8 SDK

---

## 🛠 Встановлення та налаштування

### 1. Встановлення бекенду (.NET 8)
Для роботи серверної частини необхідно встановити **.NET 8 SDK**. Його можна завантажити з офіційного сайту Microsoft .NET.

Для Windows можна використати швидке встановлення через термінал:
```bash
winget install Microsoft.DotNet.SDK.8
```
Перевірити успішність встановлення:

```bash
dotnet --version
```

### 2. Налаштування Python та залежностей
Для парсингу MAVLink-файлів та математичних розрахунків кінематики польоту проєкт використовує Python. Встановіть необхідні бібліотеки через pip:

```bash
pip install pandas numpy pymavlink
```

### 3. Встановлення фронтенду
Клієнтська частина побудована на базі React 19 та збирача Vite. У проєкті використовуються:
- React Three Fiber для 3D-візуалізації
- Lucide React для іконок та елементів інтерфейсу
- i18next для системи локалізації

Всі залежності прописані у файлі package.json. Щоб їх встановити, перейдіть до папки клієнта та виконайте команду:

```bash
cd react-app
npm install
```

## 🚀 Запуск проєкту
Примітка: Для нормальної роботи додатку обидва термінали (бекенд і фронтенд) мають працювати паралельно.

Запуск фронтенду:
Після запуску додаток буде доступний у браузері за локальною адресою, яку вкаже Vite.

```bash

```
Запуск бекенду:

```bash
dotnet run
```

## 🧮 Теоретичне обґрунтування математичних перетворень
### Кватерніони vs Кути Ейлера
Для розрахунку та візуалізації орієнтації ракети у просторі використовуються кватерніони, а не кути Ейлера. Це запобігає появі gimbal lock — втраті одного ступеня свободи при досягненні кута 90°. Кватерніони забезпечують плавну та стабільну інтерполяцію рухів ракети у 3D.

### Обробка даних IMU
Проєкт уникaє подвійного інтегрування сирих даних акселерометра для визначення позиції. Замість цього використовується гібридний підхід:
- Високочастотні дані IMU застосовуються для визначення орієнтації та згладжування швидкості.
- Дані GPS використовуються для фіксації абсолютної позиції, щоб уникнути накопичення похибки.

### Визначення точки посадки
Для переведення географічних координат у локальну метричну систему фіксується точка посадки після проходження апогею за такими критеріями:
- Падіння висоти нижче стартової відмітки (жорстка посадка).
- Зниження вертикальної та горизонтальної швидкостей майже до нуля (м'яка посадка).
- GPS координати трансформуються у декартову систему (X, Y) відносно стартового майданчика через формулу гаверсинуса (Haversine). Це дозволяє точно відтворити та візуалізувати траєкторію польоту.
